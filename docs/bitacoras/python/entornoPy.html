* PARA TENER LAS DEPENDENCIAS SEPARADAS ES BUENA PRACTICA USAR ENTORNOS VIRTUALES PARA LOS PROYECTOS
(Una librería es un producto (código listo para usar) y Una dependencia es una relación: tu programa "depende" de la librería para funcionar) (Un entorno en Python es un contexto aislado y autocontenido que incluye un intérprete de Python específico, sus bibliotecas y los paquetes de terceros que necesita un proyecto particular. Su principal utilidad es evitar conflictos de dependencias entre proyectos, permitiendo que cada uno tenga su propio conjunto de bibliotecas con versiones específicas sin afectar la instalación global de Python o a otros proyectos. Se crean para gestionar las dependencias de forma limpia y organizada, haciendo que los proyectos sean más portátiles y reproducibles)
Entonces dentro de la carpeta del proyecto creamos este entorno, escribiré en la terminal lo siguiente:
python3 -m venv venv

Luego hay que activar dicho entorno virtual asi:
source venv/bin/activate

Y ya al terminar de trabajar con el proyecto se puede salir con el comando:
deactivate
DESACTIVAR EL ENTORNO VIRTUAL ayuda a mantener el sistema organizado y asegura que cualquier librería que se instale en el futuro no se mezcle con las de este proyecto


* PARA UN PROYECTO SE SE PUEDEN UTILIZAR LIBRERÍAS DENTRO DE ENTORNOS VIRTUALES, COMO POR EJEMPLO
pygame [esto nos permitirá crear ventanas, dibujar formas, imágenes, manejar eventos de teclado o mouse, reproducir sonidos...]
numpy [esto nos permitirá hacer procesamientos numéricos, manejar matrices/arreglos eficientemente, hacer rápidas operaciones matemáticas...]
pygame usa numpy para algunos efectos de pixeles y transformaciones de imagenes.

OK luego de esa breve explicación procedemos a instalarlos DENTRO de nuestro entorno virtual nos aseguramos de tener el entorno virtual activado (si lo llamamos venv) con:
cd IA
source venv/bin/activate

Nos debemos asegurar de que pip este actualizado en este entorno
pip install --upgrade pip

Luego instalamos numpy
pip install numpy

Después ya instalamos pygame (aunque puede que se requiera instalar otros paquetes del sistema que son dependencias de SDL (Simple DirectMedia Layer, una biblioteca de bajo nivel escrita en C que proporciona acceso a hardware de multimedia como audio, teclado, ratón y gráficos, permitiendo crear aplicaciones multiplataforma. ) y otros componentes gráficos. Para Manjaro las dependencias tipicas son sdl2, sdl2_image, s dl2_mixer, sdl2_ttf...) así con pacman:
sudo pacman -Syu
sudo pacman -S sdl2 sdl2_image sdl2_mixer sdl2_ttf

Y ya con eso se puede proceder a instalar pygame con pip
pip install pygame

* para gestionar dependencias en proyectos de Python
se debe crear el archivo requirements.txt de la siguiente manera
1. asegurarnos de estar en la direccion del proyecto
2. abrir la terminal ahí
3. activar el entorno virtual con el comando [source venv/bin/activate]
4. ejecutar el comando para generar el archivo requirements.txt con el comando [pip freeze > requirements.txt] [el signo > redirige la salida del comando pip freeze [que lista todas las librerías] a un nuevo archivo llamado requirements.txt]
5. desactivar el entorno virtual al terminar con el comando [deactivate]


*PARA PROBAR EL CODIGO
cuando alguien clone tu repositorio (que ya contiene el archivo requirements.txt), el proceso para que la aplicación funcione es muy sencillo y estandarizado:
clonar el repositorio: el usuario descarga tu código desde el repositorio de Git
git clone https://github.com/...

crear un entorno virtual: el usuario crea su propio entorno virtual para evitar conflictos con otras librerías de su sistema
python3 -m venv venv

activar el entorno virtual: el usuario activa el entorno
source venv/bin/activate

instalar las dependencias: el usuario utiliza el archivo requirements.txt para que pip instale todas las librerías necesarias con las versiones correctas
pip install -r requirements.txt

una vez que este proceso está completo, el usuario tiene un entorno de trabajo idéntico al que hayas creado, con todas las librerías necesarias (como pygame y numpy) instaladas y listas para ejecutar tu código. Esto asegura que el proyecto funcione de la misma manera para todos
